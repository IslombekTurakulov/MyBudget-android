Политика конфиденциальности

Дата последнего обновления: 3 мая 2025 г.

1. Термины

  • Пользователь — лицо, использующее приложение.
  • Персональные данные — сведения, позволяющие идентифицировать Пользователя.
  • Администратор — разработчик и оператор серверной части приложения на базе Yandex Cloud.
  • Обработка данных — любые действия с данными (сбор, хранение, изменение, удаление и др.).

2. Сбор и виды данных

Приложение собирает следующие категории информации:

  • Регистрационные данные (имя, email, пароль). Пароли хранятся в зашифрованном виде с использованием алгоритма bcrypt.
  • Финансовые операции (сумма, дата, категория, описание). Для распознавания текста на чеках используется Google ML Kit или Cloud Vision.
  • Технические данные (идентификатор устройства, версия ОС, версия приложения, логи работы). Логи отправляются в Firebase Crashlytics для анализа ошибок.
  • Настройки уведомлений (предпочтения push-уведомлений через Firebase Cloud Messaging, история отправленных напоминаний).

3. Цели обработки

  • Аутентификация и контроль доступа с использованием JSON Web Tokens (JWT).
  • Синхронизация данных между устройствами через REST API на Ktor.
  • Обеспечение целостности данных с помощью ACID-транзакций в PostgreSQL.
  • Анализ и улучшение работы приложения на основе анонимной статистики в Firebase Analytics.

4. Правовые основания

  • Договор: выполнение обязательств по предоставлению сервиса.
  • Согласие Пользователя: выражено при регистрации.
  • Законные интересы Администратора: поддержание безопасности и улучшение качества сервиса.

5. Сроки хранения

  • Персональные данные — хранятся до удаления аккаунта и далее до 5 лет для юридических нужд.
  • Логи и технические метрики — хранятся в течение 1 года в Firebase.

6. Передача и раскрытие данных

Персональные данные не передаются третьим лицам, за исключением:

  • Обязательных требований законодательства.
  • Доверенных сервисов для выполнения функций (Google, Firebase, Yandex SMTP) — только необходимые объёмы данных.

7. Безопасность

  • Шифрование каналов передачи: TLS 1.3 по протоколу HTTPS.
  • Хранение паролей с bcrypt (хэш + соль).
  • Ограниченный доступ к базе данных и регулярный аудит безопасности.
  • Использование Room для локального кеширования и обеспечения офлайн-доступа.

8. Права пользователя

  • Получить копию всех своих персональных данных.
  • Внести изменения или уточнить данные.
  • Отозвать согласие на обработку данных в любой момент.
  • Запросить удаление аккаунта через службу поддержки; удаление выполняется в течение установленного срока.

9. Кэш и аналитика

  • Локальный кэш через Room обеспечивает работу в офлайн-режиме.
  • Анонимная аналитика в Firebase Analytics улучшает пользовательский опыт без привязки к личности.

10. Изменения в Политике

Все изменения публикуются не менее чем за 14 дней до вступления в силу. Уведомления доступны в приложении.
