Privacy Policy

Last updated: May 3, 2025

1. Terms

  • User — any individual using the application.
  • Personal Data — information that can identify the User.
  • Administrator — the developer and operator of the backend, hosted on Yandex Cloud.
  • Data Processing — any operation performed on data (collection, storage, modification, deletion, etc.).

2. Data Collection and Types

The application collects the following categories of data:

  • Registration data (name, email address, password). Passwords are encrypted using the bcrypt algorithm.
  • Financial transactions (amount, date, category, description). Receipt images are processed via Google ML Kit or Cloud Vision for OCR.
  • Technical data (device identifier, OS version, app version, usage logs). Logs are sent to Firebase Crashlytics for debugging.
  • Notification settings (preferences for push notifications via Firebase Cloud Messaging, history of sent reminders).

3. Purposes of Processing

  • Authentication and access control using JSON Web Tokens (JWT).
  • Data synchronization between devices via REST API built with Ktor.
  • Ensuring data integrity with ACID transactions in PostgreSQL.
  • Improving application performance based on anonymous analytics collected with Firebase Analytics.

4. Legal Basis

  • Contract: performance of obligations to provide the service.
  • User Consent: given during registration.
  • Legitimate Interests: ensuring security and improving service quality.

5. Data Retention

  • Personal Data — retained until account deletion and for up to 5 years afterward for legal purposes.
  • Logs and technical metrics — stored for up to 1 year in Firebase.

6. Data Sharing and Disclosure

Personal data is not shared with third parties except in the following cases:

  • Legal requirements by applicable law.
  • Trusted services necessary for functionality (Google, Firebase) — only essential data is shared.

7. Security Measures

  • Data transmission is encrypted using TLS 1.3 over HTTPS.
  • Passwords are stored hashed with bcrypt (hash + salt).
  • Access to the database is restricted and regularly audited.
  • Local caching using Room for offline access.

8. User Rights

  • Request and receive a copy of all personal data.
  • Request corrections or updates to personal data.
  • Withdraw consent to data processing at any time.
  • Request account deletion via support; deletion will be completed within the specified timeframe.

9. Cache and Analytics

  • Local caching via Room enables offline functionality.
  • Anonymous analytics in Firebase Analytics to improve user experience without identifying individuals.

10. Changes to This Policy

We will notify users at least 14 days before any changes become effective. Notifications are available within the app.
